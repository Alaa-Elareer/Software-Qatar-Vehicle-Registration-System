To ensure that the IQVR system has an organised and effective structure, we included object-oriented design principles into the class diagram. In order to avoid superfluous complexity, we gave priority to include only the classes that were required. Generalisation was used to create class hierarchies and remove duplication, as seen in the relationship between the Owner and the Vehicle, where the roles of each entity were clearly specified.  
  
To simplify and maintain the diagram's conciseness, data was represented as attributes within the classes. In order to control access to the class properties and preserve data integrity, getter and setter methods were implemented. As an illustration, the Insurance class controls information pertaining to insurance policies while maintaining internal data integrity via restricted access techniques.  
In order to properly illustrate links, associations between classes were constructed. There were directed arrows and multiplicities.
